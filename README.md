# Precision Patcher

Ever needed to add a new security check to 36 specific PHP files, but realized using `sed *.php` would break the rest of your server? Same here. 

**Precision Patcher** is a lightweight, fault-tolerant shell system designed for surgical code modifications. It bridges the gap between dangerous wildcard commands and tedious manual editing. 

Whether you need to swap a single string or inject a 40-line payload of code before a specific anchor line, this tool does it safely, automatically, and leaves a clean audit trail. 

### Why use this?
* **Zero Wildcards:** It only touches the exact files you list.
* **Built-in Safety Nets:** Automatically creates timestamped backups (e.g., `filename.php.20231024_1030.bak`) before modifying *anything*.
* **Fault-Tolerant:** If one file fails due to bad permissions, the script logs the error and keeps going. It won't crash your workflow.
* **Native & Fast:** Built entirely on standard Linux tools (`bash`, `sed`, `awk`). No extra packages or dependencies required.

---

##  The 5-File Architecture

To keep things clean and scalable, the system separates logic, configuration, payload, and history:

1. **`orchestrator.sh`** - The brains of the operation. You execute this.
2. **`patcher.conf`** - Your simple `key=value` rules (What are we replacing? Are we using regex?).
3. **`target_files.list`** - The specific file paths you want to target (one per line).
4. **`payload.txt`** - Your multi-line code blocks (keeps your config file clean!).
5. **`patch_history.log`** - The success/failure audit trail generated by the script.

## ⚙️ Configuration Schema (`patcher.conf`)

Before running the script, configure your patching rules in `patcher.conf`:

| Variable | Accepted Values | Description |
| :--- | :--- | :--- |
| `ACTION` | `replace_inline`, `insert_before`, `insert_after` | Determines the patching methodology. |
| `MATCH_MODE` | `exact`, `regex` | `exact` auto-escapes special chars; `regex` allows complex patterns. |
| `REPLACE_OCCURRENCE`| `first`, `all` | Stop after the first match, or update all instances in the file. |
| `SEARCH_TARGET` | `[string]` | The anchor text or regex pattern to look for. |
| `REPLACE_WITH` | `[string]` | The replacement text (Only used for `replace_inline`). |
